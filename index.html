💬 Prompt Rehberi

AI'ya Nasıl Talimat Verilir?

İyi bir prompt (talimat), iyi bir sonuç demektir. İşte etkili prompt yazmanın sırları.

1

Net ve Spesifik Olun

❌ Kötü Prompt Bir web sitesi yap
✅ İyi Prompt Kişisel portfolyo web sitesi oluştur. Üstte bir navigasyon barı olsun, ana sayfada büyük bir karşılama bölümü, altında 3 sütunlu beceriler bölümü ve en altta iletişim formu olsun. Renk teması koyu mavi ve beyaz olsun.
2

Yapıyı Tarif Edin

❌ Kötü Prompt Blog sitesi yap
✅ İyi Prompt Kişisel blog sitesi oluştur. Şu sayfalar olsun: 1) Ana sayfa — son yazıların kartlar halinde listesi 2) Hakkımda sayfası — fotoğraf alanı ve biyografi 3) Yazı detay sayfası — başlık, tarih, içerik alanı Tasarım minimal ve modern olsun, mobilde de güzel görünsün.
3

Görsel Detaylar Verin

❌ Kötü Prompt Güzel bir site yap
✅ İyi Prompt Modern bir tasarımla kişisel site oluştur. Arka plan gradient olsun (koyu mordan laciverte). Kartların köşeleri yuvarlatılmış olsun, hafif gölge efekti olsun. Yazı fontu sans-serif, başlıklar kalın. Butonlarda hover efekti olsun.

🎨 HEX Renk Kodları Nedir?

Web'de renkler HEX kodu ile belirtilir: # işareti ile başlayan 6 haneli bir kod. Örneğin:

#2563eb Mavi #dc2626 Kırmızı #059669 Yeşil #7c3aed Mor #f59e0b Turuncu

Renk seçmek için şu siteleri kullanabilirsiniz:

  • 🎨 coolors.co — Uyumlu renk paletleri oluşturun
  • 🎯 colorhunt.co — Hazır renk paletleri keşfedin
  • 📋 htmlcolorcodes.com — Renk seçici + HEX kodları

Prompt'ta şöyle kullanın: "Ana renk #2563eb (mavi), arka plan #f8fafc (açık gri), butonlar #059669 (yeşil) olsun."

🔤 Font (Yazı Tipi) Seçimi

Doğru font seçimi sitenizin profesyonel görünümü için çok önemlidir. Web'de en yaygın font kaynağı Google Fonts'tur — tamamen ücretsiz ve tüm tarayıcılarda çalışır.

  • 🔤 Google Fonts — 1.500+ ücretsiz font. En popülerler: Inter, Roboto, Open Sans, Poppins, Montserrat, Nunito
  • ✍️ El Yazısı Fontları — Davetiye, özel sayfalar için: Dancing Script, Great Vibes, Pacifico
  • 💻 Monospace Fontları — Kod gösterimi için: JetBrains Mono, Fira Code, Source Code Pro

Prompt'ta şöyle kullanın: "Başlıklarda Google Fonts'tan 'Poppins' fontu kullan (kalın 700), paragraflar için 'Inter' (normal 400). Genel yazı boyutu 16px, başlıklar 32px olsun."

Farklı fontları denemek için: fontpair.co (uyumlu font çiftleri) ve typescale.com (yazı boyutu oranları)

📐 Boşluk ve Düzen (Layout) Kavramları

AI'ya boşluk ve düzen konusunda detay verirseniz çok daha iyi sonuçlar alırsınız. İşte sık kullanılan terimler:

  • Padding (İç boşluk): Bir kutunun içeriği ile kenarı arasındaki boşluk. "Kartların içinde 24px padding olsun"
  • Margin (Dış boşluk): Elemanlar arasındaki boşluk. "Bölümler arasında 60px boşluk olsun"
  • Grid (Izgara): İçeriği sütunlara bölme. "3 sütunlu ızgara düzeninde göster, mobilde tek sütun olsun"
  • Flexbox: Elemanları yatay veya dikey hizalama. "Logo ve menü yan yana, aralarında boşluk olsun"
  • Container (Kapsayıcı): İçeriğin maksimum genişliği. "İçerik alanı maksimum 1200px genişliğinde, ortada olsun"

Prompt'ta şöyle kullanın: "Her bölümde 80px dikey padding olsun. Kartlar arasında 24px boşluk (gap) olsun. İçerik max-width 1140px ile ortalansın. Mobilde padding 20px'e düşsün."

📱 Responsive (Duyarlı) Tasarım

Web sitenizin telefon, tablet ve masaüstünde farklı boyutlara uyum sağlamasıdır. AI araçlarına "mobil uyumlu olsun" demeniz yeterlidir.

📱 Tablet 2 sütun düzen
🖥️ Masaüstü Çok sütunlu düzen
💡 Piksel mi, düzen mi?

Günümüz telefonları fiziksel olarak 1080p, 1440p hatta daha yüksek çözünürlüğe sahip. Ancak tarayıcılar "CSS pikseli" denen mantıksal bir birim kullanır — örneğin 1080px fiziksel genişliğe sahip bir telefon, tarayıcıda ~360-414px olarak görünür. Bu yüzden responsive tasarımda önemli olan piksel sayısı değil, içeriğin farklı ekran boyutlarına uyum sağlamasıdır. AI aracınıza "mobil uyumlu ve responsive olsun" demeniz yeterlidir.

⚠️ file:/// ile Açmak Ne Zaman Yetmez?

Basit HTML + CSS + JS projelerinde dosyayı çift tıklayıp açmak gayet yeterlidir. Ancak şu durumlarda yerel bir sunucu (Live Server veya Python sunucusu) gerekir:

  • fetch() ile veri çekme: JavaScript'te fetch("veri.json") gibi dosya okuma işlemleri file:/// protokolünde CORS hatası verir. Sunucu gerekir.
  • ES Modülleri (type="module"): Modern JavaScript modülleri (import/export) sadece HTTP üzerinden çalışır.
  • Service Worker / PWA: Offline çalışma özellikleri http:// veya https:// gerektirir.
  • API çağrısı yapan projeler: Harici API'lerden veri çeken sayfalar sunucu olmadan çalışmayabilir.

🐍 Python ile hızlı sunucu: Terminal/Komut satırında proje klasörünüze gidin ve şu komutu çalıştırın:

Terminal / Komut Satırı
python -m http.server 8000

Ardından tarayıcıda http://localhost:8000 adresine gidin. Siteniz gerçek bir sunucudaymış gibi çalışacaktır. Durdurmak için terminalde Ctrl+C basın.

Kural: Sadece HTML, CSS ve basit JS kullanıyorsanız → çift tıkla yeter. fetch(), modül veya API kullanıyorsanız → sunucu gerekir.

✏️ Kod Düzenleme

Basit Değişiklikler Kendiniz Yapın

AI'dan her küçük değişiklik için prompt yazmak yerine, bazı basit düzenlemeleri kendiniz yapabilirsiniz. İşte en yaygın 6 düzenleme:

1. Metin Değiştirme

Herhangi bir metni bulmak ve değiştirmek için editörünüzde Ctrl+H (Mac: Cmd+H) kullanın.

index.html
<!-- Eski metin -->
<h1>Hoş Geldiniz</h1>

<!-- Yeni metin -->
<h1>Merhaba, Ben Ahmet!</h1>

💡 İpucu: Ctrl+H ile "Hoş Geldiniz" yazıp "Merhaba, Ben Ahmet!" ile değiştirebilirsiniz.

2. Renk Değiştirme

CSS dosyanızda renk kodlarını bulun ve değiştirin. Renkler genellikle # ile başlar (HEX format).

style.css
/* Ana rengi değiştir */
background: #6366f1;
background: #e74c3c;

/* Yazı rengini değiştir */
color: #333333;
color: #1a1a2e;

💡 Renk seçmek için coolors.co veya htmlcolorcodes.com kullanabilirsiniz.

3. Resim Değiştirme

Resimleri değiştirmek için src özelliğindeki dosya adını güncelleyin.

index.html
<!-- Eski resim -->
<img src="images/photo.jpg" alt="Fotoğraf">

<!-- Yeni resim -->
<img src="images/profil.jpg" alt="Profil fotoğrafım">

💡 Yeni resmi aynı klasöre koyup adını güncelleyin. alt yazısı resim yüklenmezse görünür.

4. Link (Bağlantı) Ekleme/Değiştirme

<a href="..."> etiketlerinde URL'yi güncelleyin. Sosyal medya, e-posta veya telefon bağlantısı ekleyebilirsiniz.

index.html
<!-- Sosyal medya linki (X / eski Twitter) -->
<a href="https://x.com/kullaniciadi">X (Twitter)</a>

<!-- E-posta linki -->
<a href="mailto:ornek@email.com">Bana yazın</a>

<!-- Telefon linki (mobilde doğrudan arar) -->
<a href="tel:+905551234567">Beni Arayın</a>

💡 mailto: e-posta, tel: telefon, https:// web sitesi bağlantısı açar.

5. Yazı Boyutu Değiştirme

CSS'te font-size değerini bulun ve sayıyı artırıp azaltın.

style.css
/* Başlık boyutunu büyüt */
font-size: 24px;
font-size: 36px;

/* Paragraf boyutunu ayarla */
font-size: 14px;
font-size: 18px;

💡 px = piksel. Sayı büyüdükçe yazı da büyür. Genel metin için 16-18px idealdir.

6. Boşluk Ayarlama

CSS'te margin (dış boşluk) ve padding (iç boşluk) değerlerini değiştirin.

style.css
/* Dış boşluk (margin) */
margin: 10px;
margin: 30px;

/* İç boşluk (padding) */
padding: 8px 16px;
padding: 16px 32px;

💡 margin elementin dışındaki, padding içindeki boşluktur. Değerler: üst sağ alt sol sırasıyla.

🧪 Mini Pratik Görev

Aşağıdaki 3 küçük değişikliği kendi projenizde deneyin:

  1. Metin değiştirin: Sayfanızdaki başlık metnini kendi adınızla güncelleyin (Ctrl+H ile).
  2. Renk değiştirin: CSS dosyanızdaki bir # renk kodunu coolors.co'dan seçtiğiniz yeni bir renkle değiştirin.
  3. Link ekleyin: Sayfanıza bir <a href="mailto:..."> e-posta linki ekleyin.

Bu 3 görevi tamamladığınızda, AI olmadan küçük düzenlemeler yapabileceğinize güveneceksiniz! 🎉

GitHub Pages

ÖNERİLEN

GitHub Pages, statik web sitelerini ücretsiz olarak yayınlamanın en popüler yoludur. GitHub hesabınız varsa hemen kullanabilirsiniz.

1

GitHub Hesabı Oluşturun

github.com adresine gidin ve ücretsiz bir hesap açın.

💡 Kullanıcı adınızı nasıl bulursunuz?
  • Sağ üst köşedeki profil fotoğrafınıza tıklayın — açılan menüde kullanıcı adınız yazar
  • Profil sayfanıza gidin — URL'nin son kısmı kullanıcı adınızdır (github.com/kullaniciadi)
  • Yeni repo oluştururken Owner alanında kullanıcı adınızı görebilirsiniz
2

Yeni Repository (Depo) Oluşturun

GitHub'da sağ üstteki + butonuna tıklayıp "New repository" seçin. İsim olarak kullaniciadi.github.io yazın.

ℹ️ Repository ismini tam olarak kullaniciadi.github.io formatında yazmanız gerekir. Örnek: ahmet123.github.io
3

Dosyalarınızı Yükleyin

Boş bir repo oluşturduğunuzda sayfada göreceğiniz "uploading an existing file" linkine tıklayın. Tüm proje dosyalarınızı (index.html, style.css, script.js, resimler) sürükleyip bırakın.

ℹ️ Daha sonra dosya yüklemek için: Reponuza tekrar girdiğinizde bu link artık görünmez. Bunun yerine sayfadaki "Add file" butonuna tıklayıp açılan menüden "Upload files" seçeneğini kullanın.

📸 Dosyalarınızı sürükleyip GitHub'a yükleyin. "Commit changes" butonuna tıklamayı unutmayın.

4

Siteniz Yayında! 🎉

kullaniciadi.github.io formatındaki repolar için GitHub Pages otomatik olarak etkindir — ekstra ayar gerekmez. Birkaç dakika bekleyin, ardından siteniz şu adreste yayında olacak:

https://kullaniciadi.github.io
⚠️ index.html dosyası şart! GitHub Pages reponun kök dizininde (root) bir index.html arar. Bu dosya yoksa siteniz açılmaz.
  • Dosyalarınız bir klasörün içindeyse (ör. site/index.html), Settings → Pages bölümünden klasör yolunu ayarlamanız gerekir
  • En kolay yol: index.html, style.css ve script.js dosyalarını doğrudan reponun kök dizinine yüklemektir

🚀 İleri Seviye: Her Proje İçin Ayrı Repository

Birden fazla farklı web sitesi yayınlamak istiyorsanız, her biri için ayrı repo oluşturmak işlerinizi çok kolaylaştırır.

📂 Neden Tek Repo Yerine Ayrı Repolar?

Yukarıdaki rehberde kullaniciadi.github.io adında tek bir repo oluşturmayı öğrendiniz. Bu, ana siteniz (portfolyo, kişisel sayfa) için idealdir. Ancak zamanla farklı amaçlarla başka siteler de oluşturmak isteyebilirsiniz:

  • 🎨 Portfolyo siteniz
  • 💍 Bir düğün davetiye sayfası
  • 📝 Bir blog
  • 📊 Bir etkinlik veya proje tanıtım sayfası
  • 🎮 Bir web oyunu veya deneme projesi

Bunların hepsini tek kullaniciadi.github.io reposuna koymak, dosyalar karıştığı için bakımı ve güncellemeyi çok zorlaştırır. GitHub'ın web arayüzünden dosya yüklediğinizde, hangi dosyanın hangi siteye ait olduğunu takip etmek içinden çıkılmaz hâle gelir.

💡 Altın kural: Her farklı amaçlı web sitesi için ayrı bir GitHub repository'si oluşturun. Böylece her proje birbirinden bağımsız kalır, güncelleme ve bakım çok daha kolay olur.

🔀 İki Yaklaşımın Karşılaştırması

Tek Repo (kullaniciadi.github.io) Proje Bazlı Ayrı Repolar
Adres kullaniciadi.github.io kullaniciadi.github.io/repo-adi
Dosya düzeni Tüm projeler iç içe, karışabilir Her repo sadece kendi dosyalarını içerir
Güncelleme Yanlış dosyayı değiştirme riski Sadece o projenin dosyalarını görürsünüz
Silme / Arşivleme Klasörleri tek tek silmek zor Repoyu silin veya arşivleyin — temiz!

📋 Adım Adım: Proje Bazlı Repo ile Site Yayınlama

1

Yeni Repo Oluşturun

GitHub'da "+" → "New repository". Repo adını projenize göre verin, örneğin: dugun-davetiye, blog, etkinlik-2025.

"Public" seçin ve "Add a README file" kutucuğunu işaretleyin.

2

Dosyalarınızı Yükleyin

"Add file" → "Upload files" ile projenin dosyalarını yükleyin. Ana dosya yine index.html olmalı.

📁 İki farklı dosya konumu seçeneğiniz var:

Seçenek A — Kök dizin (root): Dosyaları doğrudan repoya yükleyin. Pages ayarlarında kaynak olarak / (root) seçin.

Seçenek B — /docs klasörü: Repoda bir docs klasörü oluşturup site dosyalarını oraya koyun. Pages ayarlarında kaynak olarak /docs seçin. Bu yöntem, repo kökünde README ve diğer proje dosyalarını ayırmanızı sağlar — daha düzenlidir.
3

GitHub Pages'i Aktifleştirin

Repo ayarlarına gidin: Settings → Pages → Branch: main seçin.

Klasör olarak dosyalarınızın konumuna göre / (root) veya /docs seçin → "Save".

4

🎉 Siteniz Yayında!

Birkaç dakika içinde site şu adreste yayınlanacak:

https://kullaniciadi.github.io/repo-adi

Örneğin repo adınız dugun-davetiye ise:
https://kullaniciadi.github.io/dugun-davetiye

📁 /docs Klasörü Ne İşe Yarar?

Normalde web sitesi dosyalarınızı (index.html, style.css, vb.) doğrudan repo köküne yüklersiniz. Ancak /docs klasörünü kullanmanın avantajları vardır:

  • 📄 Repo kökünde README.md dosyası ile projenizin açıklamasını yazabilirsiniz
  • 📂 Web sitesi dosyaları /docs içinde düzenli ve ayrı durur
  • 🧹 Proje notlarınız, tasarım dosyalarınız gibi şeyler ile yayınlanan site dosyaları birbirine karışmaz
Örnek repo yapısı (/docs kullanımı)
dugun-davetiye/
├── README.md          ← Proje açıklaması (yayınlanmaz)
├── notlar.txt         ← Kendi notlarınız (yayınlanmaz)
└── docs/
    ├── index.html     ← Ana sayfa (yayınlanır ✅)
    ├── style.css      ← Stiller (yayınlanır ✅)
    ├── script.js      ← JavaScript (yayınlanır ✅)
    └── images/
        └── foto.jpg   ← Görseller (yayınlanır ✅)
⚠️ Dikkat: /docs klasörünü kullanıyorsanız, GitHub Pages ayarlarında klasörü /docs olarak seçmeyi unutmayın! Aksi hâlde siteniz 404 hatası verir.
💡 GitHub'da klasör nasıl oluşturulur? Web arayüzünde doğrudan klasör oluşturamazsınız. Bunun yerine "Add file" → "Create new file" seçin ve dosya adına docs/index.html yazın — GitHub otomatik olarak docs klasörünü oluşturur. Ardından diğer dosyaları "Add file" → "Upload files" ile docs klasörüne yükleyebilirsiniz.

🗂️ Önerilen Repo Stratejisi

Ana siteniz → kullaniciadi.github.io reposu (portfolyo, CV, kişisel sayfa)

Her ek proje → Ayrı repo (dugun-davetiye, blog, proje-tanitim, etkinlik-2025, vb.)

Bu şekilde ana siteniz kullaniciadi.github.io adresinde, diğer projeleriniz ise kullaniciadi.github.io/repo-adi adreslerinde yayınlanır. Her biri birbirinden tamamen bağımsızdır — birini güncellemek, silmek veya arşivlemek diğerlerini hiç etkilemez.

Tüm projeleriniz aynı kullaniciadi.github.io altında olduğu için, ana sitenizden diğer projelerinize kolayca link verebilirsiniz:

🔄 Frontend & Backend

Frontend vs Backend: Farkı Anlayın

Web sitelerinin iki temel katmanı vardır. AI araçları ile yaptığımız siteler genellikle sadece frontend'tir — ve bu çoğu proje için yeterlidir!

🎨 Frontend (Ön Yüz)

HTML CSS JavaScript

Kullanıcının gördüğü ve etkileşime girdiği her şey. Tarayıcıda çalışır.

  • 📄 Sayfanın yapısı ve içeriği
  • 🎨 Tasarım, renkler, animasyonlar
  • 🖱️ Buton tıklamaları, menüler
  • 📱 Mobil uyumluluk
  • 📝 Form arayüzleri
⇄ API / HTTP

⚙️ Backend (Arka Yüz)

Node.js Python Veritabanı

Sunucuda çalışan, kullanıcının görmediği kısım. Veri işleme ve depolama.

  • 🗄️ Veritabanı işlemleri
  • 🔐 Kullanıcı girişi ve güvenlik
  • 📧 E-posta gönderme
  • 💳 Ödeme işlemleri
  • 📊 Veri analizi ve raporlama

🍽️ Restoran Benzetmesi

Bir restoran düşünün: Frontend = yemek salonu (müşterinin gördüğü kısım — menü, masalar, dekorasyon, garson). Backend = mutfak (müşterinin görmediği kısım — yemek pişirme, malzeme depolama, sipariş yönetimi). AI ile yaptığımız statik siteler, sadece "yemek salonu" kısmını oluşturur — ve kişisel siteler, portfolyolar, bloglar için bu fazlasıyla yeterlidir!

📄 Statik Site (Sadece Frontend)

HTML, CSS ve JavaScript dosyalarından oluşur. Sunucuda hiçbir işlem yapılmaz, dosyalar olduğu gibi kullanıcıya gönderilir.

Kişisel Site Portfolyo Blog CV Sayfası Düğün Davetiyesi Ürün Tanıtımı
  • ✅ GitHub Pages, Vercel, Netlify ile ücretsiz yayınlanır
  • ✅ Süper hızlı (CDN ile)
  • ✅ Güvenlik riski minimal
  • ✅ AI araçları ile kolayca yapılabilir

⚙️ Dinamik Site (Frontend + Backend)

Sunucuda kod çalışır, veritabanından veri çekilir. Her kullanıcıya farklı içerik gösterilebilir.

E-ticaret Sosyal Medya Online Bankacılık Forum/Yorum Sistemi Rezervasyon Sistemi
  • ⚠️ Sunucu ve veritabanı gerektirir
  • ⚠️ Daha karmaşık ve maliyetli
  • ⚠️ Güvenlik bilgisi önemli
  • ⚠️ Bakım ve güncelleme gerektirir

✅ Statik Site ile Yapabilecekleriniz

  • Kişisel web sitesi / Portfolyo Projelerinizi, hakkınızda bilgileri ve iletişim formunu gösterin
  • Blog yazıları Markdown veya HTML ile yazılar yayınlayın
  • İletişim formu (Formspree ile) Formspree.io kullanarak form verilerini e-postanıza alın
  • Yorum sistemi (Giscus ile) Giscus ile GitHub Discussions tabanlı yorum sistemi ekleyin
  • Fotoğraf galerisi & Etkinlik sayfası Düğün davetiyeleri, etkinlik bilgileri, galeri sayfaları
  • Veri görselleştirme (CSV/JSON) Statik veri dosyalarını tabloya veya grafiğe dönüştürün

❌ Statik Site ile Yapamayacaklarınız

  • Kullanıcı kayıt/giriş sistemi Gerçek authentication backend gerektirir (ancak Auth0, Firebase Auth alternatiftir)
  • Veritabanına veri kaydetme Kullanıcıların veri girip kaydetmesi sunucu gerektirir
  • E-posta gönderme Doğrudan frontend'den e-posta gönderilemez (Formspree alternatiftir)
  • Ödeme alma Stripe/PayPal entegrasyonu backend gerektirir
  • Gerçek zamanlı sohbet WebSocket sunucusu gerektirir

🔧 Backend Olmadan Çözümler

Bazı "backend gerektirir" gibi görünen işlemleri aslında ücretsiz servislerle veya tarayıcı özellikleriyle çözebilirsiniz:

💾 localStorage

Tarayıcıda küçük veriler saklayın. Tema tercihi, checklist durumu, son görüntülenen sayfa gibi. Veriler sadece o tarayıcıda kalır.

🍪 Cookies

Küçük metin verileri saklayın. Dil tercihi, kabul edilen bildirimler gibi. localStorage'a benzer ama sunucuya da gönderilir.

📬 Formspree / Formcarry

İletişim formlarınızdan gelen verileri e-postanıza yönlendirin. Backend yazmadan form işleme. Aylık 50 gönderim ücretsiz.

💬 Giscus / Disqus

Blog yazılarınıza yorum sistemi ekleyin. Giscus GitHub Discussions kullanır ve tamamen ücretsiz.

🔐 Auth0 / Firebase Auth

Kullanıcı giriş sistemi ekleyin — Google, GitHub, e-posta ile giriş. Auth0 aylık 7.000 aktif kullanıcı ücretsiz.

📊 Google Sheets API

Google Sheets'i basit bir veritabanı olarak kullanın. Form verilerini direkt bir tabloya kaydedin.
